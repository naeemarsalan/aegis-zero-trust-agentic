## Purpose

Red Hat OpenShift AI (RHOAI) provides the model serving and notebook infrastructure for the platform. In this PoC it serves two roles: hosting the inference endpoints that AI agents call (via RHOAI's model serving stack — KServe / ModelMesh), and providing the Data Science Project (`agentic-mcp` namespace) that scopes RHOAI resources such as workbenches, pipelines, and data connections to this project boundary.

## Exists or create

**RHOAI EXISTS** — version 3.4.0-ea.2 with DataScienceCluster `data-skill-factory` in READY state. Do **not** create a new `DataScienceCluster` or `DSCInitialization` CR — there must be exactly one per cluster and it already exists. Interact with RHOAI exclusively through its user-facing APIs: create `InferenceService`, `ServingRuntime`, `DataScienceProject`, and pipeline resources. The `agentic-mcp` namespace is the RHOAI Data Science Project for this PoC.

## Placement

- Cluster: **anaeem** (SNO, OCP 4.20.11)
- RHOAI operator namespace: `redhat-ods-operator`
- DataScienceCluster: `data-skill-factory` (existing, READY — do not modify)
- Data Science Project namespace: `agentic-mcp`
- Model serving Route (example): `https://<model-name>-predictor-agentic-mcp.apps.anaeem.na-launch.com` (auto-generated by KServe per InferenceService)
- Dashboard: `https://rhods-dashboard-redhat-ods-applications.apps.anaeem.na-launch.com`
- No additional DNS changes needed

## Security posture

- SPIFFE ID: `spiffe://anaeem.na-launch.com/ns/agentic-mcp/sa/<model-server-sa>` — model serving pods get SVIDs; agents authenticate to inference endpoints via Keycloak-issued tokens (token review by KServe's ModelMesh auth proxy)
- Model weights and data connections stored in S3-compatible object store; credentials injected via Vault Agent Injector or RHOAI's native data connection Secrets (acceptable for model weights — not runtime secrets)
- NetworkPolicy in `agentic-mcp`: default-deny; ingress on 8080/8443 from `mcp-gateway` and from the OCP router namespace; deny direct agent→model access (agents must go through agentgateway)
- Fail-mode: if an InferenceService is not Ready, agentgateway receives a `503` from the model route — no partial response is forwarded to the agent

## Maturity flags

- RHOAI 3.4.0-ea.2 is **early-access** — not production-supported; some KServe and model serving features may change before GA
- KServe raw deployment mode (no ServiceMesh) is the default on OCP 4.20; ModelMesh is the alternative for multi-model serving — confirm which runtime the `data-skill-factory` DSC enables before creating InferenceServices
- RHOAI pipelines (Kubeflow Pipelines v2 backend) is GA in 3.x
