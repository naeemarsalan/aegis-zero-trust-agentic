= The solution
:navtitle: The solution

[.lead]
Aegis is a zero-trust control plane that sits between any AI agent and the real
tools it calls. The agent authenticates with a *workload identity that cannot be
stolen as a credential*; reads run *as the user*; and every change is gated by a
*human approval* that mints a *self-expiring, scoped grant*.

== The two-speed model

Read freely, change only with a human in the loop.

== How a request flows

. *Authenticate the workload, not a secret.* SPIRE issues each agent a SPIFFE identity based on what it _is_ -- no API key to provision, leak, or rotate.
. *Read as the user.* The gateway swaps the agent's identity for the user's scoped token (Keycloak RFC 8693). The downstream tool sees the human.
. *Block writes by default.* A privileged action returns `403` until approved -- there is no degrade-to-allow path.
. *Approve in a console.* A human -- *not the requester* -- approves a scoped, time-boxed request. The act of approving mints the grant.
. *Use it once, then it's gone.* The grant is short-lived (write = 5 min, single-use; exec = 30 min) and auto-expires. No standing privilege, no manual revoke.

== Why it is supportable

Exactly *one* component is custom code: `ext-proc-delegation`, a small Go service
that swaps identity and injects the downstream token. Everything else is a
supported product or established open source -- SPIRE (Red Hat ZTWIM), Keycloak
(RHBK), HashiCorp Vault, Kyverno, agentgateway, OpenShift GitOps. Any
MCP-speaking agent gets these guarantees with *zero agent-side changes*.
