= UC1 -- Delegated read
:navtitle: UC1 Credential delegation

[.lead]
The agent calls a tool and never holds the tool's credential.
The downstream service sees the *user*, not the agent -- enforced structurally, not by convention.

== How the delegation works

[mermaid,target=uc1-sequence,format=svg]
....
sequenceDiagram
  autonumber
  participant Agent as "AI agent<br/>(SVID only)"
  participant SPIRE as "SPIRE<br/>Workload API"
  participant Gateway as "Gateway +<br/>ext-proc-delegation"
  participant Vault as "Vault<br/>consent grant"
  participant KC as "Keycloak<br/>RFC 8693"
  participant Tool as "pfsense-mcp<br/>(any MCP tool)"
  participant Audit as "WORM audit<br/>Loki"

  Agent->>SPIRE: fetch JWT-SVID (aud=gateway)
  SPIRE-->>Agent: SVID — cryptographic identity, no downstream secret

  Agent->>Gateway: MCP call (Bearer SVID)
  Note over Gateway: verify SVID against SPIRE OIDC JWKS — fail closed

  Gateway->>Vault: read consent grant for this sandbox
  Note over Vault: grant exists → proceed, missing → deny

  Gateway->>KC: RFC 8693 token exchange — agent actor, user subject
  KC-->>Gateway: user-scoped token (aud=tool)

  Gateway->>Tool: MCP call — user identity injected, agent SVID cleared
  Note over Tool: sees the USER, never the agent

  Tool-->>Agent: response — credential headers stripped

  Gateway->>Audit: structured event (args SHA-256 hashed)
....

== What the agent sees vs. what the tool sees

[cols="1,2,2",options="header"]
|===
| | Agent sees | Tool sees

| Identity
| Its own SVID before the call; credential-stripped response after
| User's RFC 8693 token scoped to this tool's audience

| Credential
| None -- no API key ever reaches the agent pod
| Per-tool credential fetched from Vault in-memory, discarded after one request

| Arguments
| Its own MCP JSON-RPC body (unchanged)
| Same body (ext-proc-delegation does not modify arguments)
|===

== Why the invariant holds

* *No credential in the pod.* The agent has no Secret volume mounts.
  The Vault fetch happens inside `ext-proc-delegation` for the lifetime of one request.
* *Fail closed.* If the SVID is invalid, the consent grant is missing, or Keycloak is unreachable,
  the request is denied -- there is no degrade-to-allow path.
* *Response is scrubbed.* `ext-proc-delegation` strips credential and authorization headers from
  every response before the agent sees it.
* *Every call is audited.* A structured audit event is emitted per request;
  tool arguments are SHA-256 hashed before logging -- never raw.

For the write/approval half of the model see xref:05-uc2-jit-sub-identity.adoc[UC2 JIT elevation],
or xref:11-security-model.adoc[the security model] for trust boundaries.
