= UC3 -- Models on the same identity plane (MaaS, no tokens)
:navtitle: UC3 MaaS model plane

[.lead]
The agent calls AI models authenticating with the *same SPIFFE JWT-SVID* it uses
for tools -- no model API key anywhere. Standard models need only the identity;
premium models need the identity *plus* a human-approved capability. The agent's
own *brain* runs on this plane too.

  Gateway->>LLM: STANDARD authorized
  LLM->>Up: inject upstream key (server-side)

  Agent->>JIT: request -> human approves

  Gateway->>LLM: 200 — premium model

== What this proves

* *The SVID is the model credential.* No per-agent model bearer token to mint,
  store, rotate, or leak -- the agent reuses the identity it already holds for MCP.
* *Two tiers, one gate.* STANDARD (`/openrouter`, `/`) needs a valid JWT-SVID;
  PREMIUM (`/premium/…`) needs the SVID *and* a jit-approver capability JWT -- the
  same approve-to-elevate mint-gate as a tool write.
* *The upstream key stays server-side.* For the external LLM (OpenRouter
  `claude-sonnet-4` / `claude-opus-4`), the vendor API key lives only in Vault and
  is injected by `llm-proxy` for one request -- it never reaches the agent.
* *Even the brain obeys it.* The agent's Claude brain reasons through MaaS with a
  fresh SVID per call (loopback `maas_brain_proxy.py`); `env` in the pod shows no
  model key.
