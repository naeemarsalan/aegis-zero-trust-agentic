= Business outcomes
:navtitle: Business outcomes

[.lead]
Aegis converts "we cannot let AI agents touch production" from a policy argument into
*mechanically enforced, individually auditable guarantees* -- without slowing the agent down on routine reads.

== What the platform delivers

[cols="2,3,2", options="header"]
|===
|Outcome |How Aegis delivers it |Who cares

|Agents can touch production safely
|Reads are delegated instantly as the user. Writes are blocked by default -- no change lands without a human approval.
|CISO, engineering leads

|Passes security and compliance review
|Four-eyes enforcement (approver ≠ requester), WORM audit trail. Equivalent to change-control for a non-human operator.
|Security, auditors, GRC

|No credential sprawl
|The agent pod holds only a cryptographic workload identity (SPIFFE). There are no API keys, tokens, or passwords to steal or rotate.
|Security, platform team

|Smaller breach blast radius
|A compromised agent yields nothing usable. A steered (prompt-injected) agent still cannot bypass the policy gate or skip the human approval.
|CISO, risk

|Machine-speed ops with human authority
|Reads are instant and delegated. Approvals happen in the console -- a seconds-to-minutes flow, not a change-board cycle.
|Engineering, operations

|Works across the whole toolchain
|The same control plane applies to any MCP-speaking tool: firewall, Kubernetes, cloud, database, DNS. No per-tool integration work.
|Platform team, architects
|===

== The cost model

* *One custom component* to own: `ext-proc-delegation`, a small Go service. Everything else is vendor-supported (SPIRE/ZTWIM, Keycloak RHBK, Vault, Kyverno, ArgoCD) or LF-governed OSS.
* *Latency*: one token exchange plus one Vault read per delegated tool call -- sub-second on the read path.
* *Process change*: privileged agent actions wait for a console approval. That wait is the control working as designed.
