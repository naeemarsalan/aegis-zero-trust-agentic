== What each persona does

[cols="2,3,3", options="header"]
|===
|Persona |What they do |What they see

|Requesting engineer
|Gives the agent a task in plain language; monitors progress.
|Chat interface or CLI. Never handles credentials or approvals.

|AI agent
|Executes reads immediately (delegated as the user); requests human approval before any write.
|Only its own SPIFFE workload identity. No stored tokens, keys, or passwords.

|Approver
|Reviews a scoped write request in the Approval Console and clicks approve or deny. Must be a different person than the requester.
|Approval Console: the verb, resource, scope, and TTL being requested. One click to approve; one to deny.

|Security / compliance reviewer
|Verifies that every privileged action has a human behind it; traces requests from audit log back to user identity and approval record.
|WORM audit trail (Loki / Grafana): who asked, who approved, what ran, when the grant expired.

|Platform team
|Deploys and upgrades the identity stack via GitOps; defines which tools are reachable and what policies apply.
|ArgoCD, operator CRs, this Git repository. See xref:40-gitops.adoc[GitOps & bootstrap].
|===
