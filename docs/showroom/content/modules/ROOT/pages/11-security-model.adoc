= Security model
:navtitle: Security model

[.lead]
Every hop is authenticated, fail-closed, and WORM-audited.
A compromised agent pod cannot escalate privilege, cannot hold a downstream credential,
and cannot forge an approval.

The *credential frontier* runs through TB-B: credentials exist only inside that boundary
(in `ext-proc-delegation` memory and in Vault).
They never cross into TB-A (the agent), never appear in TB-C as the agent's credential,
and never enter git or etcd.

== Threat controls at a glance

[cols="1,2",options="header"]
|===
| Threat | Control

| *Stolen credential (prompt-injection / pod compromise)*
| Agent holds only a SPIFFE SVID -- a cryptographic identity assertion, not a usable downstream secret.
  Nothing to steal.

| *Forged or replayed identity*
| SVID is SPIRE-issued, validated against SPIRE OIDC JWKS.
  Subject is bound to the exact namespace/service-account at issuance.

| *Fail-open on error*
| Both gateway filters (authz + mutation) are required.
  Any timeout, network error, or parse failure returns a denial -- never a pass-through.

| *Agent self-approves a write*
| The Approval Console enforces approver ≠ requester at the mint layer, not by policy convention.
  The agent's SVID cannot call the mint endpoint.

| *Grant outlives its window*
| Capability JWTs are structurally short-lived (write = 5 min single-use; exec = 30 min).
  Expiry is enforced at the gate -- no cron, no manual revoke needed.

| *Audit tampered or missing*
| Every gateway decision and every approval emits a structured WORM event to Loki.
  Tool arguments are SHA-256 hashed before logging -- never raw.

| *Lateral movement via default-allow network*
| Default-deny NetworkPolicy; only explicitly allowed routes between namespaces exist.
  Kyverno enforces the allow/deny decision before any credential is minted.
|===

== What "proven" means here

The full journey is verifiable by inspection:

* Pod has no Secret volumes -- agent pod carries only the SPIFFE CSI socket.
* Upstream logs show the user subject, not the agent SPIFFE ID.
* Response to the agent carries no Authorization or credential header.
* WORM audit in Loki ties every call to actor, scope, outcome, and SHA-256-hashed arguments.

See xref:04-uc1-credential-delegation.adoc[UC1] for the delegated read,
xref:05-uc2-jit-sub-identity.adoc[UC2] for the approval flow,
